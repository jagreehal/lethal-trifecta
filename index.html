Common Injection Techniques
Technique How It Works
Hidden HTML/CSS Instructions in font-size:0 or display:none elements — invisible to humans, visible to LLMs
Document metadata Injections embedded in EXIF data, PDF annotations, or file properties
Authority assertion "IMPORTANT SYSTEM UPDATE" or "maintenance mode" to impersonate the system
Fake completion Pre-completed responses that trick the model into thinking the task is done
Payload splitting Splitting malicious instructions across multiple fields that recombine during processing
Tool/function abuse Coercing the model to invoke tools with attacker-chosen arguments
Retrieval poisoning Planting injections in RAG knowledge bases so the model cites attacker content as truth
Defense Patterns (from Evertz et al.)
Pattern Description
Action-Selector A constrained selector (not the LLM) chooses which tool to invoke, preventing the LLM from being tricked into calling exfiltration tools
Plan-Then-Execute The LLM generates a plan before seeing untrusted content; execution follows the plan without re-prompting
Dual LLM A privileged LLM handles tools and private data; a quarantined LLM processes untrusted content with no tool access
Code-Then-Execute The LLM writes code that is statically analyzed before execution, preventing dynamic injection
Context-Minimization Reduce the amount of untrusted content in the LLM context to shrink the attack surface
